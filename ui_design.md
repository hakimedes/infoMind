# InfoMind - UI 功能设计文档

> **版本**: v1.0  
> **数据来源**: 参考自 `prd.md`  
> **核心定位**: 个人化、沉浸式、结构化的知识可视化系统。

---

## 1. 整体布局架构 (Layout)

系统采用 **“极简导航栏 + 动态核心工作区”** 的布局结构，以确保用户的视野聚焦于知识内容本身。

### 1.1 顶栏 (Navbar)
顶栏始终固定在页面顶端，提供全局操作入口：
- **左侧 (Brand)**：应用 Logo 及名称（InfoMind）。
- **中间 (Search)**：全局搜索框，支持快捷键 `⌘K` 唤醒。搜索输入时通过防抖实时呈现结果。
- **右侧 (Actions)**：
  - 📈 **统计报表 (Stats)**：跳转至全局大盘视图（热力图/统计表）。
  - ➕ **快捷添加 (Quick Add)**：点击展开 URL 录入小面板，可以快速粘贴链接并备注。
  - ⚙️ **系统设置 (Settings)**：呼出设置模态面板。

### 1.2 过滤器栏 (Filter Bar)
悬浮于主工作区上方，用于动态筛选信息流：
- **分类胶囊 (Category Pills)**：显示如 `[全部]`, `[自然科学]`, `[历史]` 等，带有前置 Lucide 极简图标。
- **排列组合 (Sort & Views)**：
  - 支持排序选项（最新优先等）。
  - 支持在不同视图间一键切换（书架模式 🔲 / 时间线模式 ☰）。

## 2. 核心视图模式 (View Modes)

系统提供多维度的知识展现形式，满足沉浸式回顾与快速检索的不同场景。

### 2.1 书架模式 (Bookshelf View) - 默认视觉
以“书房书架”为隐喻，是最具沉浸感的核心展现形式。
- **层级划分**：按预定义的 25 个行业分类分层展现（每个分类占据一行或多行虚拟木质或分隔线条）。
- **分类头部**：每层顶部展示分类图标（Lucide 格式）+ 分类名称 + 该分类下拥有的内容总数。
- **图书渲染 (Book Card)**：
  - **封面元素**：抓取的网页/视频原图作为正封面，左侧增加 CSS 3D 折痕/阴影形成明显的“书脊”立体感。
  - **动态角标**：内容若为同一作者的系列文章（被大模型聚合成的一本书），右上角显示 `(N篇)` 角标。
  - **平台烙印**：右下角通过 Logo/主色调隐约标识来源平台（如：YouTube红色，B站蓝色）。
  - **文字排版**：封面下方展示精简后的标题及原作者。

### 2.2 时间线模式 (Timeline View)
瀑布流与时间轴结合的形式，适合复盘近期的收集轨迹。
- 按收录时间（倒序）以列表/卡片形式贯穿。
- 侧边附带清晰的日期刻度线展示。

### 2.3 搜索结果模式 (Search View)
当搜索框触发焦点且具有内容时，主界面临时隐藏，全屏覆盖显示搜索结果的精准列表，重点高亮关键词及所在分类。

## 3. 模态与弹窗交互 (Modals & Drawers)

### 3.1 书籍详情面板 (Book Detail Modal)
当用户点击某一本“书”时，弹出抽屉或居中模态框。
- **全幅展示区**：展示更清晰的封面图及完整的书籍副文本（作者、大模型聚合的书名、收录时间等）。
- **标签流 (Tags)**：列出大模型自动提取的 3-5 个核心关键词。
- **智能摘要区 (AI Summary)**：通过 Markdown 格式展示这段/多段知识内容的 100-200 字长文本要点提炼。
- **集合目录 (Index)**（仅限聚合型书籍）：列出这本“合集”中所有的原始链接及各自日期。
- **操作按钮**：提供原链跳转 (`[🔗 查看原文]`)、手动修正 (`[✏️ 编辑分类]`)、删除等按钮。

### 3.2 快捷收录栏 (Quick Add Bar)
从顶部 Navbar ⬇ 呼出的下拉带状条，提供低信噪比的输入体验：
- 包含极简的 URL 输入框及一个备选的 Note 文本框。
- 提交后，提供实时的**动态状态反馈**（加载动画 -> 解析成功提示音/Toast）。

### 3.3 系统设置面板 (Settings Modal)
承载底层配置和偏好调整：
- **大模型配置区**：切换供应商 (OpenAI 兼容 / Anthropic)，输入 Base URL, 动态密码掩码形式保护 API Key，输入 Model。
- **状态联动**：自带 `[🧪 测试连接]` 按钮，并直观提示反馈延迟毫秒数。
- **展示偏好/其他**：深浅色主题切换及基础视图数据统配。

## 4. 视觉与交互规范 (UI/UX Principles)

1. **降噪设计**：摈弃强感官刺激色彩，背景采用低对比度色调。减少 Emoji 大面积滥用，改用 **Lucide 极简几何图标体系**（如 `<i data-lucide="brain"></i>`），增强文艺与科研感。
2. **拟物与扁平的平衡 (Neumorphism / 3D Hints)**：不在整体 UI 动刀，仅在“书本”实体上做深层次的 3D 书脊和阴影高光投射处理，还原翻阅书本的质感。
3. **异步加载反馈**：在所有请求（如 LLM 分析中、链接拉取中）统一运用非阻断的优雅动画（Spinning loaders / Skeleton screens），并在完成后使用状态柔和的通知吐司 (Toast)。
4. **空状态引导 (Empty States)**：当初建无内容，或搜索结果为空时，需呈现友好的插画与引导操作提示。
